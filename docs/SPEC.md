# AutoSub Media Player — Product & Technical Spec v1

## Context
Greenfield app (repo: `shmulious-student/autosub-media-player`). A cross-platform
**macOS + iOS** media player that ingests a media library, builds contextual data
per title (a "character bible": characters, genders, name translations,
relationships), and plays video with high-quality, correctly-timed subtitles
generated by **local, free, commercially-licensed AI models**, translating into a
user-selectable target language (**default Hebrew** → RTL + gendered-grammar
correctness is the product's core differentiator). The Mac does the heavy AI; the
iPhone is a light player + sync client.

---

## 1. Product overview

**What it is:** Plex-like personal media player whose killer feature is
**AI-generated, context-aware translated subtitles** that get the *gender,
grammar, and character-name consistency* right — pre-generated in the background
so playback is instant.

**Who does what:**
- **macOS app** = the hub. Scans the library, enriches metadata, builds bibles,
  runs ASR + translation, stores subtitle artifacts, serves them to iOS.
- **iOS app** = light companion. Browses the library, plays video (streaming
  from Mac/NAS on LAN, or offline-pinned downloads), shows the synced subtitles.

## 2. Locked decisions

| Area | Decision |
|---|---|
| Target language | User-selectable, **default Hebrew**; RTL + gender/grammar correctness first-class |
| Playback model | **Pre-process first** (background transcribe+translate before play); live = later phase |
| Subtitle source | **User chooses per title**: translate embedded sub track, or ASR from audio (smart default + override) |
| Character bible | **Hybrid** (AI proposes characters/genders/relationships; user reviews & corrects) |
| Bible scope | Shared at **series + franchise** level (one bible per "contextual parent") |
| Compute model | **Mac is the hub** (all heavy AI on Mac); iOS stays light |
| Tech stack | **Flutter** app shell + **media_kit/libmpv** playback; **native Mac engine** for AI |
| Library source | **Local folders + network shares (SMB/NAS)** |
| Metadata | **TMDB** (API key available) for titles/cast/character names; filename-parse fallback |
| Translation engine | **Context-aware local LLM** consuming the bible |
| Licensing | **Commercial** → permissive models only (Whisper MIT; **DictaLM 3.0 Apache** for Hebrew; **Qwen3/Qwen-MT Apache** for other langs). No NLLB/Aya/GPL |
| Sync | **iCloud (remote) + Bonjour/LAN (fast)**; only artifacts sync, not video |
| Processing flow | **Auto-queue new imports**; user can reprioritize/pause/manually trigger |
| Subtitle display | **Translated-only** default (dual-sub = later) |
| Subtitle storage | **Portable sidecar (.srt/.ass) next to media + internal indexed copy** |
| iOS video access | **LAN stream (SMB) + explicit offline pin/download**; remote streaming = later |

## 3. Validated technology choices

- **Playback:** `media_kit` + **libmpv playback-only commercial build** (universal
  MKV/HEVC on macOS + iOS). **Never enable FFmpeg `--enable-gpl`**; decode/demux
  only (LGPL-safe); ship attributions; **avoid libVLC and x264/x265**. Keep an
  `AVPlayer` fast path for plain MP4/H.264 on iOS; wire `AVAudioSession` +
  background-audio + Now Playing manually.
- **ASR:** **WhisperKit** (MIT, ANE-accelerated, Swift, word-level timestamps) as
  primary, with a **forced-alignment refinement pass** (WhisperX-style, ±50 ms
  target) since timing accuracy is critical. whisper.cpp behind the same interface
  as fallback.
- **Translation LLM (Mac):** **DictaLM 3.0** (Apache-2.0, Hebrew-specialized;
  GGUF) for the Hebrew default; **Qwen3 / Qwen-MT** (Apache-2.0, 100+ langs) for
  other targets. Runtime: **MLX** preferred on Apple Silicon, **llama.cpp (GGUF)**
  as portable backend, behind one inference interface (12B for speed / 24B for
  quality tier).
- **Engine bridge:** **local sidecar daemon** (Swift binary in the `.app`) over a
  **loopback HTTP/gRPC + event stream**, *not* embedded FFI — because jobs are
  long-running, need crash isolation, a persistent queue, and push progress. The
  same daemon hosts the Bonjour LAN server. Player uses libmpv in-process.
- **Store:** **SQLite** on the Mac = source of truth; a derived read-only library
  index is what syncs to iOS.
- **Model storage:** all heavyweight weights + download caches live on the
  **external drive** at `/Volumes/EP2TB/autosub-models/` — never the internal disk
  or `~`. The engine resolves paths from `$AUTOSUB_MODELS` and fails loudly if the
  drive is unmounted. See [MODELS.md](MODELS.md).

## 4. End-to-end pipeline (Mac engine, 12 stages)

```
Scanner → MetadataEnrich(TMDB) → Grouper(contextual parent) → BibleBootstrap
→ SourceSelector(embedded sub | ASR) → Demux/AudioExtract → ASR(WhisperKit+align)
→ Segmenter(CPS/reading-speed) → Translator(bible-aware LLM) → Assembler(.srt/.ass, RTL)
→ Store(sidecar + SQLite) → SyncServer(iCloud + Bonjour)
        └─ persistent JobQueue: priority, pause/resume, crash-recoverable ─┘
```

**Quality mechanisms:**
- **Bible→LLM injection:** per segment, pass only the relevant characters
  (canonical name, **gender**, target name-translation, relationships, speaker).
- **Gender correctness (Hebrew):** resolve **speaker + addressee** per line (sub
  speaker tags / dialogue heuristics / LLM speaker-attribution pass) and feed both
  genders to the model; DictaLM as the Hebrew-native engine.
- **Glossary lock:** character name translations & honorifics pinned across a
  contextual parent so they never drift between episodes/films.
- **Timing:** word-level timestamps → line boundaries on speech pauses; enforce
  **CPS** + min/max duration (Hebrew CPS tuned lower than English); embedded-sub
  path keeps original timings, re-flows translated text to fit.
- **Correction loop:** user edits a character → bible `version` bumps → artifacts
  with older `bible_version_used` for that parent are invalidated & re-queued
  (line-level where possible); `user_corrected` entries are never AI-overwritten.

## 5. Data model (core entities)
- **Title** — path, content_hash, container/codec, duration, `contextual_parent_id`, `tmdb_id`, `source_preference(embedded|asr|auto)`, status.
- **ContextualParent** — type(series|franchise|standalone), `tmdb_id`, `bible_id`.
- **CharacterBible** — `contextual_parent_id`, `version`, `locked_by_user`.
- **Character** — `canonical_name`, `gender(m|f|nb|unknown)`, `name_translations{lang→str}`, aliases[], relationships[], confidence, `user_corrected`.
- **SubtitleArtifact** — `title_id`, lang, format(srt|ass), source, engine+model+version, `sidecar_path`, `internal_blob_id`, cps_stats, qa_flags[], `bible_version_used`.
- **ProcessingJob** — title_id, stage, state(queued|running|paused|failed|done), priority, progress, attempts, error.
- **SyncRecord** — artifact_id, transport(icloud|lan), state, checksum, device_targets[].

---

## 6. Sync & iOS video
- **Syncs (small):** library index, character bibles, subtitle artifacts — via
  **Bonjour/LAN** (home, fast) and **CloudKit/iCloud** (remote). Mac is
  authoritative; bibles are the only iOS-editable entity → per-record versioning +
  last-writer-wins with merge prompt.
- **Video (large) never syncs.** iOS plays via **LAN SMB streaming** from Mac/NAS
  with the local synced subtitle, plus **explicit offline pin** to download a title
  for on-the-go. Remote video streaming is a later phase.

---

## 7. Phased delivery
- **v0 — vertical slice (Mac only):** scan one folder → pick one MKV → WhisperKit
  ASR → DictaLM Hebrew translation with a hand-stubbed 3-character bible → `.srt`
  sidecar → play in media_kit with RTL Hebrew sub. Proves the whole chain.
- **v1 — usable Mac product:** TMDB enrich, contextual-parent grouping, AI bible
  bootstrap + **review/correction UI**, source selection, persistent **job queue**
  (auto/pause/reprioritize), sidecar+SQLite store, forced-align timing, CPS.
- **v2 — iOS companion:** light iOS player, **Bonjour LAN sync** of
  index+bible+sidecars, **LAN SMB streaming** + offline pin.
- **v3 — iCloud + scale:** CloudKit remote sync, correction→re-translate polish,
  multi-target-language switching (Qwen path), batch perf tuning.
- **v4 — later:** live/on-the-fly processing; on-device iOS ASR; dual subtitles.

---

## 8. Top risks
1. **Hebrew gender/grammar correctness** is the core promise & least measurable →
   build an in-house EN/src→HE gender+RTL eval set early; gate model choice on it.
2. **Timestamp drift** degrades timing → forced-alignment as a first-class stage.
3. **iOS HEVC/4K** may soft-decode on older devices → device-matrix test; AVPlayer
   fast path.
4. **(L)GPL + App Store** friction → playback-only libmpv, no GPL flags, attributions.
5. **Pre-processing throughput** (hours for big libraries) → persistent queue,
   overnight batch, model-size tiers.
6. **Bible invalidation storms** on franchise edits → line-level re-translate +
   debounce + confirm scope.

## 10. Verification (per phase)
- **v0:** run the macOS app on a real MKV; confirm a timed, RTL, gender-correct
  Hebrew `.srt` is produced and renders in the player. Spot-check gender lines.
- **v1:** import a multi-episode series; confirm one shared bible, AI-proposed
  characters editable in UI, a correction re-queues affected subs; queue
  pause/reprioritize works.
- **v2:** on a second device, confirm bible+sidecars sync over LAN, video streams
  via SMB, and an offline-pinned title plays with no network.
- Throughout: in-house Hebrew gender/RTL eval set scored against each model/tier.
