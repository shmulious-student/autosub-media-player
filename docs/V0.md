# v0 — vertical slice (Mac only)

This document describes the **v0 skeleton** that lives in the repo today and the
TODO checklist to turn it into a working vertical slice. The authoritative
blueprint is [SPEC.md](SPEC.md); model-storage rules are in [MODELS.md](MODELS.md).

> v0 goal (SPEC §7): scan one folder → pick one MKV → WhisperKit ASR → DictaLM
> Hebrew translation with a hand-stubbed 3-character bible → write a `.srt`
> sidecar → play in `media_kit` with the RTL Hebrew subtitle. Proves the whole
> chain on Mac.

## Architecture

Two processes, talking over loopback HTTP:

```
┌──────────────────────────┐        127.0.0.1:8765        ┌───────────────────────────┐
│  Flutter app (lib/)      │  ─── HTTP + SSE progress ──▶ │  Mac engine (engine/)     │
│                          │                              │   Swift sidecar daemon    │
│  • HomePage              │                              │                           │
│  • LibraryPage (grid)    │  ◀── library index / jobs ── │  12-stage PipelineOrch.   │
│  • PlayerPage            │                              │  JobQueue (in-mem stub)   │
│    media_kit + RTL .srt  │                              │  ASRService (WhisperKit)  │
│  • EngineClient (HTTP)   │                              │  BibleAwareTranslator     │
│  • SyncProvider stubs    │                              │  Store (SQLite, stub)     │
└──────────────────────────┘                              │  DaemonServer (loopback)  │
                                                          └───────────────────────────┘
                                                              models: $AUTOSUB_MODELS
                                                              (external drive only)
```

- The app does **no AI**. It renders the library, drives the player, and talks to
  the engine via `EngineClient` (`lib/engine/engine_client.dart`).
- The engine owns the pipeline, queue, ASR, translation, store, and the loopback
  server. Heavy model weights are resolved from `$AUTOSUB_MODELS` and the engine
  **fails loudly** if the external drive is unmounted (`ModelPaths.swift`).
- Playback is in-process in the app via `media_kit`/libmpv (playback-only,
  LGPL-safe build — never a GPL FFmpeg).

```

## How to run

Set the model storage env first (or use the Makefile, which exports it):

```sh
export AUTOSUB_MODELS=/Volumes/EP2TB/autosub-models
```

**Engine:**

```sh
make engine-build      # swift build
make engine-test       # swift test
make engine-run        # starts the stub daemon; fails loud if drive unmounted
```

**App:**

```sh
make app-get           # flutter pub get
make analyze           # flutter analyze
make app-test          # flutter test
make app-run           # flutter run -d macos
```

## What compiles today

- **Flutter:** `flutter analyze` clean, widget test passes. `media_kit`, `drift`
  (+ `sqlite3_flutter_libs`), `path_provider`, `http` all resolve.
- **Engine:** `swift build` green, `swift test` green (3 tests). The daemon runs
  and performs the real `$AUTOSUB_MODELS` mount check.

Everything beyond the data model + prompt builder is a **stub with a TODO**.

## TODO checklist → working v0 vertical slice

1. **Embed an HTTP server in the engine** (e.g. Swifter) and bind `DaemonServer`
   to `127.0.0.1:8765`; implement `POST /jobs`, `GET /library`, `GET /health`,
   and the `GET /jobs/{id}/events` SSE progress stream. Wire `EngineClient` to
   the real endpoints.
2. **Demux/AudioExtract stage:** extract a 16 kHz mono WAV from the chosen MKV
   (libmpv/ffmpeg demux-only, no GPL flags).
3. **WhisperKit ASR:** add the WhisperKit dep, point its model folder at
   `modelPaths.whisperKit`, transcribe with word-level timestamps, add the
   forced-alignment refinement pass (±50 ms).
4. **Segmenter:** word timestamps → subtitle lines on speech pauses, enforce
   Hebrew-tuned CPS + min/max duration.
5. **Hand-stub the 3-character bible** (canonical name, gender, `he` name
   translation) and feed it into `BibleAwareTranslator` per line (speaker +
   addressee).
6. **DictaLM translation:** add llama.cpp (GGUF) or MLX, load DictaLM 3.0 from
   `modelPaths.llm`, run `buildPrompt` output, return the translated line only.
7. **Assembler + Store:** write a UTF-8 RTL `.srt` sidecar next to the media and
   record a `SubtitleArtifact`; persist the `InMemoryStore` to SQLite.
8. **App playback:** resolve the produced sidecar path in `LibraryPage`, pass it
   to `PlayerPage`, confirm the RTL Hebrew sub renders and timing is correct.
9. **Spot-check gender lines** (SPEC §10 v0 verification) and ship attributions
   for the libmpv/FFmpeg build.
