# v0 — vertical slice (Mac only)

This document describes the **v0 skeleton** that lives in the repo today and the
TODO checklist to turn it into a working vertical slice. The authoritative
blueprint is [SPEC.md](SPEC.md); model-storage rules are in [MODELS.md](MODELS.md).

> v0 goal (SPEC §7): scan one folder → pick one MKV → WhisperKit ASR → DictaLM
> Hebrew translation with a hand-stubbed 3-character bible → write a `.srt`
> sidecar → play in `media_kit` with the RTL Hebrew subtitle. Proves the whole
> chain on Mac.

## Architecture

Two processes, talking over loopback HTTP:

```
┌──────────────────────────┐        127.0.0.1:8765        ┌───────────────────────────┐
│  Flutter app (lib/)      │  ─── HTTP + SSE progress ──▶ │  Mac engine (engine/)     │
│                          │                              │   Swift sidecar daemon    │
│  • HomePage              │                              │                           │
│  • LibraryPage (grid)    │  ◀── library index / jobs ── │  12-stage PipelineOrch.   │
│  • PlayerPage            │                              │  JobQueue (in-mem stub)   │
│    media_kit + RTL .srt  │                              │  ASRService (WhisperKit)  │
│  • EngineClient (HTTP)   │                              │  BibleAwareTranslator     │
│  • SyncProvider stubs    │                              │  Store (SQLite, stub)     │
└──────────────────────────┘                              │  DaemonServer (loopback)  │
                                                          └───────────────────────────┘
                                                              models: $AUTOSUB_MODELS
                                                              (external drive only)
```

- The app does **no AI**. It renders the library, drives the player, and talks to
  the engine via `EngineClient` (`lib/engine/engine_client.dart`).
- The engine owns the pipeline, queue, ASR, translation, store, and the loopback
  server. Heavy model weights are resolved from `$AUTOSUB_MODELS` and the engine
  **fails loudly** if the external drive is unmounted (`ModelPaths.swift`).
- Playback is in-process in the app via `media_kit`/libmpv (playback-only,
  LGPL-safe build — never a GPL FFmpeg).

## Audio: decode, don't transcode

We never write a per-video intermediate WAV. `AudioDecoder` stream-decodes the
chosen audio track straight out of the ORIGINAL container (MKV, MP4, …) into
in-memory 16 kHz mono float PCM via libav (ffmpeg), so every common
container/codec works out of the box — AVFoundation alone can't open MKV. This is
**decode only** (no GPL encoders, no `--enable-gpl`); for shipping we bundle an
LGPL libav build rather than the system binary.

## How to run

Set the model storage env first (or use the Makefile, which exports it):

```sh
export AUTOSUB_MODELS=/Volumes/EP2TB/autosub-models
```

**Engine:**

```sh
make engine-build      # swift build
make engine-test       # swift test
make engine-run        # starts the stub daemon; fails loud if drive unmounted
```

**App:**

```sh
make app-get           # flutter pub get
make analyze           # flutter analyze
make app-test          # flutter test
make app-run           # flutter run -d macos
```

## Run the v0 slice (CLI)

The `process` subcommand runs the back half of the pipeline on one file. Generate
the deterministic fixture first (macOS `say` + ffmpeg):

```sh
python3 scripts/make_test_fixture.py      # → fixtures/sample.mkv + transcript

swift run --package-path engine AutoSubEngine \
  process fixtures/sample.mkv \
  --transcript fixtures/sample.transcript.json --target he
# → fixtures/sample.he.srt  (RTL, gender-correct, timed)
```

With `--transcript`, lines flow through the **real** `BibleAwareTranslator`
interface (speaker/addressee gender + locked glossary prompt) but use the
fixture's reference Hebrew — so the player path is verifiable before the LLM is
wired. Without `--transcript`, the production path runs (WhisperKit + DictaLM,
currently stubs → tasks below).

## What works today

- **Decode → segment → translate(iface) → RTL .srt** runs end-to-end on the
  fixture: AAC-in-MKV decoded in memory, cues timed within CPS budget, Hebrew
  gender correct for speaker (1st person) and addressee (2nd person).
- **Engine:** `swift build`/`swift test` green. **Flutter:** `flutter analyze`
  clean, widget test passes.

Still stubbed: real ASR, real LLM, the HTTP daemon, SQLite persistence, sync.
